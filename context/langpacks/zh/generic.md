# 通用语言包

本包设定了适用于所有语言的跨语言行为期望。

## 错误/正确示例 1 — 魔法数字

**错误**
```text
timeout = 37
retry = 5
```

**正确**
```text
const DEFAULT_TIMEOUT_SECONDS = 30
const MAX_RETRY_ATTEMPTS = 5
```

使用魔法数字会模糊意图并使未来修改变得困难。命名常量保留含义并减少出错风险。

## 错误/正确示例 2 — 显式错误处理

**错误**
```text
result = doWork()
// ignore error
```

**正确**
```text
result, err = doWork()
if err != nil:
    return err
```

静默失败会隐藏问题并导致级联错误。始终显式处理错误并闭环。

## 通用规则

- 保持命名与现有代码一致。
- 偏好显式错误处理而非隐式失败。
- 避免魔法数字；使用命名常量。
- 保持函数小而单一职责。
- 分离关注点：解析、验证、执行和 IO。
- 最小化副作用；显式传递依赖。
- 在引入新模式之前先匹配现有模式。

## 指导

使用解决问题的最小修改。不确定时，搜索代码库中类似的模式并遵循它们。避免在没有明确需求的情况下添加新的抽象层。保持输出简洁并与现有惯例一致。
